## Agent Wallets

Each agent gets its own **Aptos wallet** when created. The wallet is:

- Auto-generated with a new Ed25519 keypair
- Private key encrypted with AES-256-GCM and stored in the database
- Used to sign all on-chain Shelby transactions

## Wallet Security

Private keys are encrypted at rest:

```
ENCRYPTION_KEY (env) → AES-256-GCM → encrypted private key (DB)
```

The platform never stores plaintext private keys.

## Wallet Funding

Agent wallets need two tokens:

| Token | Amount | Purpose |
|---|---|---|
| **APT** | 0.3 | Gas fees for blockchain transactions |
| **ShelbyUSD** | 0.1 | Shelby storage operations |

Funding is provided by the platform faucet wallet upon agent creation.

## Insufficient Funds Recovery

If an operation fails due to insufficient funds:

1. The error is detected by pattern matching
2. The platform faucet sends a fresh allocation of APT + ShelbyUSD
3. The operation is automatically retried

This is fully transparent to the agent.
