---
title: "Top Up Credits"
description: "Initiate and confirm credit purchases via cryptocurrency payments"
---

## Step 1: Initiate Top-Up

```
POST /v1/billing/initiate-topup
Authorization: Bearer <JWT>
Content-Type: application/json
```

Creates a payment request. Provide either `credits` (integer) or `amountUsd` (number).

### Request Body

```json
{ "credits": 1000 }
```

or

```json
{ "amountUsd": 10.00 }
```

### Response

```json
{
  "amountUsd": 10.00,
  "credits": 1000,
  "creditPriceUsd": 0.01
}
```

Use `amountUsd` to configure the Tender SDK payment widget with a matching `amount` and a unique `referenceId`.

---

## Step 2: Confirm Top-Up

```
POST /v1/billing/confirm-topup
Authorization: Bearer <JWT>
Content-Type: application/json
```

After the Tender SDK's `onEventResponse` callback fires with a `completed`, `partial-payment`, or `overpayment` status, call this endpoint to credit the account.

### Request Body

```json
{
  "paymentId": "unique-reference-id",
  "sdkStatus": "completed",
  "sdkAmount": 10.00
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `paymentId` | `string` | ✅ | The unique reference ID used with the Tender SDK |
| `sdkStatus` | `string` | ✅ | Payment status from the SDK callback (`completed`, `partial-payment`, `overpayment`) |
| `sdkAmount` | `number` | ✅ | USD amount from the SDK's `onFinishResponse.data.amount` |

### Success Response (credited)

```json
{
  "status": "credited",
  "paymentStatus": "completed",
  "usdReceived": 10.00,
  "creditsAdded": 1000,
  "creditBalance": 1850
}
```

### Pending Response (not yet credited)

```json
{
  "status": "not_credited",
  "paymentStatus": "pending",
  "message": "Payment is pending. Credits will be added when payment completes.",
  "creditBalance": 850
}
```

## Errors

| Status | Reason |
|---|---|
| `400` | Missing `paymentId` or `sdkStatus` |
| `401` | Missing or invalid JWT token |
| `409` | Payment already processed (idempotent) |
| `422` | `sdkAmount` missing, zero/negative, or exceeds $10,000 cap |
| `503` | Payment gateway not configured |

<Warning>
  Each `paymentId` can only be confirmed once. Duplicate calls return `409` with the current balance.
</Warning>
